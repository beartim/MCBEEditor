# Blocktopograph iOS 13 v1.1.7（117）修改说明

## 1. 新建实体按世界实际格式存储

- 修复旧式世界中新建或复制实体被固定写成 `actorprefix/digp`，导致游戏、`testfor @e` 与 `kill` 无法识别的问题。
- 新建前检查当前世界的区块 `Entity(0x32)`、`ActorDigestVersion`、有效 `digp/actorprefix` 以及复制来源，自动选择旧版或现代存储。
- `ActorDigestVersion` 作为现代世界的强标志；若世界仍有区块 Entity 且没有该标志，即使旧版程序曾误写 `digp/actorprefix`，仍按旧式格式创建。
- 复制来源是区块 Entity 时，目标实体继续追加到目标区块的 `Entity(0x32)` 连续 NBT。
- 修复复制旧实体时将 Short/Int 数字 `id` 错写为字符串的问题；同类复制保留原数字 ID 与类型，改成其他已知实体时按数据值表更新数字 ID。
- 旧式世界空白新建实体时直接创建数字 `id`；无法映射数字 ID 的自定义实体会明确拒绝，而不是写入游戏不可识别的记录。
- 现代 Actor 世界继续使用 `actorprefix<UniqueID>` 与对应区块 `digp`，不退回旧格式。
- 新建完成提示会显示实际采用的存储来源。

## 2. 旧版数字方块 NBT 编辑

- 旧版 SubChunk 方块详情不再只读。
- NBT 编辑器显示 `legacy_id`、`legacy_data` 和 `name`。
- 支持把数字 ID 修改为 0–255、数据值修改为 0–15，并重新编码旧版 SubChunk。
- 保留原 SubChunk 版本、Y 索引语义和方块数据后的尾随记录。
- 阻止把现代 NBT 调色板状态混写入旧版数字 ID SubChunk。

## 3. 方块坐标渲染中心

- 方块坐标模式使用输入坐标对应方块中心 `X+0.5/Z+0.5` 作为视口锚点。
- 不再先换算为区块坐标再使用区块中心，负坐标同样保持精确。
- 区块坐标模式仍维持原来的区块中心行为。
