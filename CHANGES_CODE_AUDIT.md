# MCBEEditor 1.0.0 代码审计与稳定性优化

本次不改变应用版本，仍为 **1.0.0（构建号 100）**。审计覆盖地图、实体/方块实体、NBT 编辑器、世界文件事务、导入解压、通知刷新和测试脚本。

## 地图与对象扫描

- 取消“无限缩放时按完整范围建立平方级临时数组”的实现，改为覆盖完整逻辑范围的自适应代表区块采样。
- 普通范围仍逐区块精确渲染；超大范围最多解码有限数量的代表区块，并将结果按实际覆盖范围映射到画布，避免内存爆炸和整数溢出。
- UIKit 画布尺寸归一化到安全范围；界面显示的有效缩放倍率与内部画布倍率分离，防止极端倍率产生超大视图或图层。
- 实体和方块实体从两次数据库扫描合并为一次；大范围改用全库索引后按坐标过滤，避免对大量不存在区块逐个查询。
- HardcodedSpawners 在超大范围改为扫描数据库键索引，小范围继续使用直接区块键读取。
- 普通 NBT、方块、实体等内部修改只清理缓存并原位刷新，不再错误重置当前维度、地图中心、缩放、图层和选择状态；只有外部整体替换存档才完整重置。
- 删除已被现有入口取代且没有调用的旧地图函数和旧预取计算。

## NBT 与实体代码整理

- 将八处重复的 NBT 可见树遍历、展开和搜索逻辑合并到 `NBTTreeRows`，所有编辑器使用同一实现。
- 删除旧实体删除、旧区块记录删除、旧村民数量/ID 推断等已被新逻辑完全取代的私有函数。
- 清理应用源码中的可避免强制解包与强制类型转换；实体范围描述和村庄边界推断改为安全解包。
- 实体大范围扫描增加安全的范围乘法、坐标溢出检查和匹配对象上限语义，防止“前面不匹配的对象耗尽上限”。

## 世界文件事务

- 删除世界改为暂存事务：先把世界和元数据移动到 `.DeletionStaging`，索引提交成功后再删除；提交失败自动回滚。
- 应用启动时自动恢复“索引仍存在但文件停留在暂存区”的中断删除，并清理已提交删除遗留的暂存目录。
- 世界重命名在写入前保存 `level.dat` 和 `levelname.txt` 的精确快照；任一步骤或索引保存失败时恢复原文件和原索引名称，避免半完成状态。
- Application Support 路径增加 Documents 和临时目录回退，不再依赖强制取得第一个系统目录。

## 导入安全

- ZIP 解压在写文件前统一验证全部路径，拒绝目录穿越、绝对路径、驱动器路径、重复路径和大小写/Unicode 规范化冲突。
- 增加单文件解压大小、总解压大小和异常压缩比限制，降低 ZIP bomb 和磁盘耗尽风险。
- 解压后继续校验长度与 CRC；所有文件仍采用原子写入。

## 测试与维护

- 新增自适应地图采样、共享 NBT 树、WorldStore 中断删除恢复和审计约束测试。
- 回归脚本会检查旧地图死代码没有重新出现、各 NBT 编辑器继续使用共享遍历、关键安全实现存在且应用源码没有 `try!`、`as!`、`first!`、`last!`。
- 所有 Swift 源文件通过 `swiftc -parse`，并通过 `swift-format lint`。
- 完整核心测试按模块分段执行通过；单个总脚本在当前 Linux 环境会因大型 Swift 测试编译超过单次工具时限而中止，不代表用例失败。
